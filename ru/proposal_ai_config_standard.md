# Предложение: стандартизировать каталоги AI-инструментов

Чтобы уменьшить количество каталогов в `$HOME` и обеспечить предсказуемую
структуру для AI-инструментов, предлагается простая конвенция на основе
директорий XDG.



## Конфигурация

```text
~/.config/ai/
```

## Данные

```text
~/.local/share/ai/
```

Каталог `ai` выступает общей корневой точкой для AI-экосистемы. Каждый
инструмент хранит свои файлы в отдельном подкаталоге.

## Пример структуры

```text
~/.config/ai/
├── prompts/               # общие промпты
├── providers/             # определения провайдеров
├── mcp/                   # конфигурации MCP
├── <etc>/
└── app/
    ├── aichat/
    ├── codex/
    ├── claude/
    ├── cli/               # ai-cli
    └── <etc>/

~/.local/share/ai/
├── prompts/
├── cache/
├── memory/
├── <etc>/
└── app/
    ├── aichat/
    ├── codex/
    ├── claude/
    ├── cli/
    └── <etc>/
```

## Преимущества

* Уменьшает количество скрытых каталогов в `$HOME`
* Предоставляет единое и предсказуемое расположение AI-ресурсов
* Упрощает резервное копирование и перенос между машинами
* Отделяет общие ресурсы экосистемы от данных конкретных инструментов
* Создаёт основу для будущей совместимости без необходимости вводить
  сложные спецификации

Предложение определяет только структуру каталогов и соглашения об именовании.
Форматы файлов и внутренняя организация данных остаются на усмотрение
разработчиков конкретных инструментов.
