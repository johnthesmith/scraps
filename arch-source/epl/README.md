# Онтология Entity-Property-Link (EPL) модели.

## Правила

1. Каждая сущность описана в своём YAML-файле.
2. Наименование `name` содержит максимально короткое уникально имя сущности или 
аббревиатуру.
3. Описание `description` содержит максимально короткое описание сущности или 
расшифрову аббревиатуры, не использует собственное имя сущности.
4. Ссылки в описании — только на сущности того же или вышестоящего уровня (через `id`).
5. Не следует плодить сущности сверх необходимого (бритва Оккама).
